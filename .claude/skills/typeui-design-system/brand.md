## Rules

- Use the project name above for branded product labels, empty states, auth screens, and AI-generated project copy.
- Use the logo asset as the primary brand mark; do not add initials or unrelated text inside the icon.
- Brand color variables in this file override conflicting values in `colors.md`.
- Use Brand for primary actions, important links, active states, focus emphasis, and identity moments.
- Use Secondary and Accent only as supporting colors for highlights, charts, illustrations, and small affordances.
- Use Surface for page, section, card, and input background derivation.
- The brand kit may override name, logo, typography reference, and colors only.
- The brand kit must not override component anatomy, spacing, radius, shadow, density, motion, or interaction rules from the attributed design skill.
- Pair logo-only usage with accessible text labels where the mark alone would be ambiguous.
